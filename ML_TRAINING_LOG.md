# 互动度预测模型 — 机器学习训练日志

## 1. 项目背景

目标：基于歌曲的特征（播放量、时长、发行年份等），预测一首歌的评论数，进而量化「互动度」。

### 业务意义
一首歌的热度（播放量）和互动（评论数）不一定成正比。有的歌「听的人多但没人评论」，有的歌「虽不太热但评论区很活跃」。通过 ML 模型预测预期评论数，实际值/预测值 = **互动度**。

| 互动度 | 含义 |
|--------|------|
| > 1.2 | 高互动：粉丝愿意花时间留言 |
| 0.7 ~ 1.2 | 正常互动 |
| < 0.7 | 低互动：只听不评 |

---

## 2. 数据采集

数据来源：网易云音乐 API → MySQL

- 批量爬取 80+ 位歌手（华语流行、R&B、民谣、古风、外语等混合）
- 每位歌手取 Top 50 热门歌曲
- 每首歌字段：`plays`, `comments_count`, `duration`, `publish_year`, `artist_id`
- 关联歌手特征：`music_size`（总歌曲数）、`album_size`（总专辑数）

### 数据量统计

| 阶段 | 数量 |
|------|------|
| 原始爬取 | ~2000+ 条 |
| 清洗后 | 视实际过滤结果 |
| 训练集 | 80% |
| 测试集 | 20% |

---

## 3. 特征工程

### 原始特征

| 特征 | 类型 | 含义 |
|------|------|------|
| `plays` | 数值 | 歌曲热度值 (0-100) |
| `duration_min` | 数值 | 歌曲时长（分钟） |
| `era` | 数值 | 距今多少年 = 2026 - publish_year |
| `music_size` | 数值 | 该歌手总歌曲数 |
| `album_size` | 数值 | 该歌手总专辑数 |
| `log_plays` | 数值 | log(plays+1) 对数变换 |

### 特征工程步骤

1. **清洗**：过滤 plays=0 的数据，过滤年份异常（<1980或>2026）
2. **单位转换**：duration 毫秒→分钟
3. **对数变换**：log_plays = log(plays+1)，使分布更正态
4. **时间特征**：era = 2026 - year，更直观
5. **目标变量**：log_comments = log(comments+1)，回归目标

### 为什么用 log 变换？

评论数分布极度偏斜（几千条 vs 几条），直接预测原始值会受离群点主导。log 变换后分布更均匀，模型更容易学到规律。

## 4. 模型选型

对比了两种模型：

### 模型 A: RandomForestRegressor（最终选择）

- 适用非线性和复杂交互关系
- 不易过拟合
- 自带特征重要性分析

### 模型 B: LinearRegression（基线）

- 仅用 `plays` 一个特征
- 作为对比基准

## 5. 超参数调优（GridSearchCV）

使用 **5折交叉验证 + GridSearch** 搜索最佳参数组合。

### 搜索空间

| 参数 | 候选值 | 搜索总数 |
|------|--------|---------|
| `n_estimators` | 100, 200, 300 | 3 |
| `max_depth` | 10, 20, None | 3 |
| `min_samples_split` | 2, 5, 10 | 3 |
| `min_samples_leaf` | 1, 2, 4 | 3 |
| **总组合** | | **81 种** |

### 调参过程

```
GridSearchCV(cv=5, scoring='r2', n_jobs=-1)
对每种参数组合做 5 折交叉验证
→ 选择平均 R² 最高的参数
→ 用最优参数重新训练
```

### 调参结果

搜索完成后从 `ml_result.json` 中读取 `best_params` 字段查看。

---

## 6. 评估指标

| 指标 | 含义 | 理想值 |
|------|------|--------|
| **R²** | 模型解释力，1 为完美 | 越高越好 |
| **MAE** | 预测 vs 实际的平均偏差（评论数） | 越小越好 |
| **RMSE** | 对大误差更敏感的偏差 | 越小越好 |

### 基线对比

```
基线模型（仅用 plays）R² vs 完整模型 R²
→ 差值 = 添加其他特征带来的提升
```

## 8. 结果解读

### 互动度计算

```
互动度 = 实际评论数 / 模型预测的预期评论数
```

- **互动度 > 1.2**：这首歌的评论区异常活跃，粉丝粘性强
- **互动度 0.7~1.2**：正常，播放量和评论数相当
- **互动度 < 0.7**：听的人多但没人评论，可能只是「背景音乐」

### 偏差点分析

模型预测偏差大的样本最有分析价值：
- 预测偏低 → 实际评论多 → 说明粉丝活跃度高
- 预测偏高 → 实际评论少 → 说明歌好听但粉丝不爱评论

---

## 9. 模型局限性

1. **特征有限**：没有歌词内容、歌手影响力、宣传力度等维度
2. **评论数仅代表数量**：没有考虑评论质量（点赞数、回复数）
3. **数据偏差**：只爬取了热门歌手的 Top 歌曲，不能代表全量

---

## 10. 结论

通过 RandomForest 回归模型，可以利用歌曲和歌手特征较准确地预测预期评论数，进而量化「互动度」。该指标可以帮助理解不同歌曲的粉丝活跃模式。
